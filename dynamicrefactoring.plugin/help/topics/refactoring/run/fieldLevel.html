Running Refactorings: Attribute-Scope Operations


Whenever you select an attribute from a class in the tree on the Package Explorer view, field-scope refactorings become available. In order to run any of them, you have three choices.

  • Right-click on the attribute you want to apply the refactoring on, and select the popup-menu item Dynamic Refactoring > Field Level Operations.


    Popup run attribute refactoring


  • Inside Eclipse select the menu item Dynamic Refactoring > Refactorings > Field Level Operations.


    Menu run attribute refactoring


Once you tell the plugin you want to run a refactoring on the selected element, it starts building the internal model it will use. When the process is over, a selection window with the available field-scope refactorings will open.


Field-scope available refactorings


It is now that you must select the refactoring you would like to apply. When you have already chosen one, push the Accept button in order for the refactoring process to start.

  • When you select a class from an Eclipse view, the "Available Reactorings" view update its contents with the field-scope refactorings. Then you have to double-click one of them to start the ejecution of one of the refactorings.


    "Available Refactorings" view
