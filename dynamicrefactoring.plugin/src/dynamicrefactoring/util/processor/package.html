Permiten manejar los elementos Java desde el punto de vista del modelo Java interno de Eclipse. En general, permite identificar cada elemento y relacionarlo con su elemento equivalente en el metamodelo MOON que maneja el plugin.

El n�cleo de org.eclipse.jdt define los tipos de elementos Java que constituyen el modelo interno Java manejado por Eclipse. Dentro del subpaquete core, se define la interfaz común a los elementos aceptados, IJavaElement.

De entre los tipos que IJavaElement define como posibles tipos que pueden tener sus objetos, resultan �tiles de cara a la integración del plugin con Eclipse los siguientes, para cada uno de los cuales existe una subinterfaz propia:

  • Java project (proyecto Java): objetos IJavaProject.
  • Package fragment root (carpeta raíz de código fuente, a partir de la que comienza la estructura de paquetes): objetos IPackageFragmentRoot.
  • Package fragment (paquetes): objetos IPackageFragment.
  • Compilation unit (fichero Java de clase): objetos ICompilationUnit.
  • Type (tipo definido por una clase o interfaz): objetos IType.
  • Field (atributo de clase): objetos IField.
  • Method (método): objetos IMethod.
  • Local variable (una variable local o un argumento formal de un método): objetos ILocalVariable.

Por cada tipo que se necesita relacionar con elementos del modelo MOON utilizado, se define una clase encargada de procesarlo. Todas proporcionan una interfaz similar, con la salvedad de JavaLocalVariableProcessor, que además debe permitir distinguir si un objeto ILocalVariable es un argumento formal o una variable local de un método.
