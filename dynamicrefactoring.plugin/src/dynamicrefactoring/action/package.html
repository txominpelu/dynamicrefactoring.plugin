Proporciona la implementación de las acciones que integran la funcionalidad del plugin en la interfaz de Eclipse.

Contiene las clases que implementan el enganche básico entre el plugin y Eclipse. Son clases que representan la funcionalidad disponible del sistema, a las que invocará Eclipse cuando se utilicen las opciones de los menús correspondientes al plugin.

La declaración de estas clases a través del fichero de definición del plugin, plugin.xml permite a Eclipse conocer qué menús y desplegables debe poner a disposición del usuario para que pueda utilizar las funciones del plugin. Se crean automáticamente sobre la interfaz dichos menús a partir de un objeto que representa la acción correspondiente a modo de proxy. El objeto real con la acción que se debe ejecutar solo se instancia una vez el usuario ha elegido la opción correspondiente; hasta ese momento, Eclipse tiene suficiente con conocer el lugar donde debe aparecer el menú, y el texto, icono, etc. que lleva asociados.

Todas las acciones mantienen una interfaz común, definida en Eclipse por IActionDelegate.

El método fundamental de las acciones es run(), invocado desde el proxy asociado cada vez que se debe lanzar la acción. Las subclases deben implementar aquí la tarea que ejecuta su acción.
