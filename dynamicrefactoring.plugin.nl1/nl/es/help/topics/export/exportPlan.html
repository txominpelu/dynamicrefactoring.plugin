Exportar un plan de Refactorizaciones

En esta sección aprenderá cómo exportar el plan de refactorizaciones que ha sido ejecutado sobre un proyecto determinado, de forma que luego pueda ejecutarse en un nuevo proyecto con las características pertinentes.

  1. Seleccione la opción de menú Refactorización Dinámica > Exportar ... para abrir el diálogo Exportar plan de Refactorizaciones....




    Seleccione Exportar plan de Refactorizaciones... en el menú


    La ubicación por defecto en la que se exportará el plan de refactorizaciones se puede establecer dentro del menú Window > Preferences de Eclipse en el apartado "Directorio de exportación para el plan de refactorizaciones" del submenú Dynamic Refactoring.



    menú Preferences


  2. Para seleccionar una ubicación destino, diferente a la establecida por defecto, en la que se desea exportar el plan de refactorizaciones se debe presionar el botón "...".



    directorio destino


  3. Una vez que se solicita la exportación, se mostrará un mensaje de confirmación indicando que la exportación se ha realizado de forma satisfactoria.



    Confirmación


    Se creará una carpeta con el plan de la refactorizaciones cuya estructura se corresponde con la siguiente:
        - Una carpeta llamada refactorings que contiene una carpeta por cada una de las refactorizaciones que conforman en plan.
        - Un fichero XML llamado refactoringPlan.xml que guarda una descripción de las refactorizaciones ejecutas.
        - Un fichero DTD llamado refactoringPlanDTD.dtd que contiene la estructura que tiene que tener el fichero refactoringPlan.xml para que pueda ser interpretado por la aplicación.
        - Una carpeta llamada repository que contiene todos los ficheros .class correspondientes a los mecanismos (acciones, precondiciones y postcondiciones) que componen las refactorizaciones del plan.


    Para cancelar la operación de exportación se puede pulsar el botón Cancelar.
